Zarys zadania

Celem zadania jest implementacja hierarchii klas, obrazujących strukturę bohaterów w grze MOBA (na przykładzie League of Legents) - tiery, lane'y oraz bohaterowie; oraz narzędzi wspomagajacych.

Hierarchia klas wygląda następująco:

  • Tier – klasa zawierająca nazwę tieru Name typu string, oraz konstruktor w postaci Tier(string name)
  • Lane – klasa dziedzicząca po klasie Tier zawierająca nazwę lane'u Name typu string, metodę GetTierName() zwracającą nazwę tieru, oraz konstruktor Lane(string Name, string _Tier) który ustawia nazwę lane'u i nazwę tieru
  • Hero – klasa dziedzicząca po klasie Lane zawierająca nazwę bohatera Name typu string, metodę GetLaneName() zwracającą nazwę lane'u, oraz konstruktor Hero(string Name, string _Lane, string _Tier) który ustawia nazwę bohatera, lane'u i nazwę tieru
  • L6 – Klasa pomocnicza

Etapy

Etap 1 (2 punkty)

Stwórz hierarchię klas, tak by spełniały założenia z zarysu zadania. Wszystkie klasy powinny przedefiniowywać metodę GetHashcode() oraz Equals(object obj)

Etap 2 (1 punkt)

Stwórz klasę L6 - jej składowe to

  • pole _Tier typu Tier zawierające aktualny tier oraz metodę SetTier(Tier Tier) która ustawia tier, nie zmieniając jej jeśli obiekt nie jest typu Tier (czyli nie typu Lane lub Hero)
  • pole Lanes typu List<Lane> zawierające aktualną listę lane'ów
  • pole Heros typu List<Hero> zawierające aktualną listę bohaterów
  • AddLane, metoda która przyjmuje jeden argument typu Lane dodaje go do listy Lanes. Metoda zwraca true jeśli lane dodano, false jeśli lane znajduje sie już na liście, null jeśli lane'u nie można dodać (element nie jest lanem, nie jest odpowiedniej tieru).
  • AddLaneIgnoreConstraints, metoda która przyjmuje jeden argument typu Lane dodaje go do listy Lanes. Metoda zwraca true jeśli lane dodano, false jeśli lane znajduje sie już na liście.

Etap 3 (2 punkty)

Rozszerz klasę L6

  • AddHero, metoda która przyjmuje jeden argument typu Hero dodaje go do listy Heros. Metoda zwraca true jeśli bohatera dodano, false jeśli bohater znajduje sie już na liście, null jeśli bohatera nie można dodać (element nie jest bohaterem, nie jest odpowiedniego tieru, na liście lane'ów nie ma odpowiedniego lane'u).
  • AddHeroIgnoreConstraints, metoda która przyjmuje jeden argument typu Hero dodaje go do listy Heros. Metoda zwraca true jeśli bohatera dodano, false jeśli bohater znajduje sie już na liście.
  • AreAllLinesDifferent - metoda która zwraca true jeśli wszystkie wszyscy bohaterowie na liście mają inny lane.

Słowniczek

  • Tier, czyli Poziom - tak bohaterowie w grach MOBA (i nie tylko) sa klasyfikowani przez graczy. Im wyższy poziom tym bohater jest uznawany za potężniejszego. Czasami używane są numerki tierów (im mniejszy tym lepszy) lub opisy słowne, np. God-Tier (najpotężniejszy) czy Trash (nie da się grać)
  • Lane - w grach MOBA takich jak League of Legends, postacie sterowane przez komputer poruszają się zazwyczaj jedną z trzech ścieżek - górną, dolną bądź centralną. Niektórzy bohaterowie sprawdzają się lepiej na konkretnej ścieżce niż na innych - w szczególności w grach profesjonalnych gdzie panują pewne niepisane zasady przydziału bohaterów do pewnych ról.
*Oczywiście, można by tez zaproponować podział bohaterów na Role zamiast Lane'y, ale to nie jest celem tego ćwiczenia.
